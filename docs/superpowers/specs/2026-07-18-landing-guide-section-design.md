# 랜딩 이용방법 스크롤 섹션 설계

- 날짜: 2026-07-18
- 상태: 설계 승인됨 (A안 — 단계 카드 리스트)

## 목적

랜딩 페이지에 서비스 이용방법 안내가 없어 처음 방문한 사용자가 두 핵심 기능(일정 조율하기, 중간 지점 찾기)의 사용 흐름을 알기 어렵다. 랜딩 첫 화면 아래에 스크롤로 노출되는 "이용방법" 섹션을 추가해, 진입 장벽 없이 자연스럽게 사용 흐름을 소개한다.

## 결정 사항

- **형태**: 랜딩 페이지 하단 스크롤 섹션 (온보딩 모달·별도 페이지·코치마크 아님)
- **콘텐츠 범위**: 일정 조율하기 / 중간 지점 찾기 두 기능 모두, 각 4단계
- **디자인**: 별도 시안 없음. 텍스트 위주 단계 카드 + 기존 캐릭터 SVG 포인트 활용. 새 라이브러리·스크롤 애니메이션 없음

## 화면 구성

### 1. 기존 첫 화면 (변경 최소화)

`src/domains/meeting/pages/index.tsx`의 로고 + 메인 버튼 2개 영역은 그대로 유지한다. 첫 화면 하단에 스크롤 유도 힌트("이용방법 보기" 텍스트 + 아래 화살표 아이콘)를 추가한다.

### 2. 이용방법 섹션 (신규)

`src/domains/meeting/components/landing-guide-section.tsx` 컴포넌트로 분리하고 `LandingPage`에서 첫 화면 섹션 아래에 렌더링한다.

구성:

- 섹션 헤더: "모여락 이용방법" 제목
- 기능별 서브 섹션 2개, 각각 소제목 + 캐릭터 SVG 포인트 + 단계 카드 4개
  - 일정 조율하기: primary 계열 컬러 (기존 `bg-primary-main` 톤과 일치)
  - 중간 지점 찾기: sub 계열 컬러 (기존 `bg-sub-main` 톤과 일치)
- 단계 카드: 번호 뱃지 + 제목 + 한 줄 설명 (텍스트 위주)

### 3. 단계 콘텐츠 (라우트 흐름 기반)

**일정 조율하기**

1. 모임 만들기 — 모임 이름과 정보를 입력해 시작해요
2. 후보 날짜 고르고 링크 공유 — 가능한 날짜 후보를 정하고 친구들에게 링크를 보내요
3. 가능한 날짜 투표 — 참여자들이 각자 되는 날짜에 투표해요
4. 모두가 되는 날 확인 — 투표 결과로 최적의 날짜를 확인해요

**중간 지점 찾기**

1. 모임 만들기 — 모임 이름과 정보를 입력해 시작해요
2. 링크 공유하고 출발지 등록 — 참여자들이 각자 출발지를 등록해요
3. 중간지점 역 추천 받기 — 모두에게 공평한 지하철역을 추천받아요
4. 주변 장소 둘러보기 — 추천 역 주변 장소를 보고 약속 장소를 정해요

문구는 구현 시 자연스럽게 다듬을 수 있으나 단계 구조(4단계 × 2)는 유지한다.

## 컴포넌트 구조

- `LandingPage` (기존) — 첫 화면 + `<LandingGuideSection />` 렌더링, 스크롤 힌트 추가
- `LandingGuideSection` (신규, `src/domains/meeting/components/landing-guide-section.tsx`) — 섹션 전체. 내부에서 기능별 데이터(제목·컬러·캐릭터·단계 배열)를 상수로 정의하고 서브 섹션을 map으로 렌더링
- 단계 카드는 별도 파일로 분리할 만큼 크지 않으면 같은 파일 내 로컬 컴포넌트로 둔다

상태·라우팅·API 변경 없음. 정적 콘텐츠만 추가한다.
